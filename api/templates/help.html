Website Functions

1. Add a server

Go to Servers and fill out:

  • Name: Friendly display name (must be unique)
  • Host: IP or DNS name
  • Port: Usually 22
  • Username: Remote Linux user
  • Auth Method: SSH Key or Password
  • Sudo Password: Optional; needed only if sudo prompts for it

Click Save Server. To modify an existing server, use Edit in the table and then Update Server.

2. Run updates now

Go to System Updates:

  • Select one or more target servers
  • Choose package manager (or Auto detect)
  • Click Start Update

Watch progress in the jobs table and open Latest job output for full logs.

3. Create cron schedules

In System Updates → Schedules:

  • Set a schedule name
  • Enter a cron expression (server time in UTC)
  • Select target servers and package manager
  • Click Create Schedule

Example cron expressions:

0 2 * * *      # daily at 02:00 UTC
0 */6 * * *    # every 6 hours
30 3 * * 0     # every Sunday at 03:30 UTC

Use Enable/Disable and Delete in the schedule table to manage existing schedules.

4. Understand failures

Dashboard and Updates tables include a Details column with a failure snippet. For full context, open Latest job output.

  • Auth failed: verify username/password or SSH key
  • sudo errors: verify sudo password or sudo rights
  • package manager unknown: host may not be apt/dnf/yum based
  • timeout/unreachable: network, DNS, firewall, or host offline
