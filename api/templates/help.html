How to Use Daygle Server Manager

1. Dashboard

The dashboard shows a quick operational snapshot:

  • Total configured servers
  • Running and failed update jobs
  • Latest update activity

2. Servers

Go to Servers to add or manage remote hosts.

  • Name: unique display name
  • Host: IP or DNS name
  • Port: usually 22
  • Username: SSH account
  • Auth Method: SSH key or password
  • Sudo Password: optional; use only when sudo prompts for a different password

Use Edit and Delete in the table for existing entries.

3. System Updates (Run Now)

In System Updates you can run immediate package updates:

  • Select one or more target servers
  • Choose package manager (Auto detect, apt, dnf, yum)
  • Click Start Update

The Update Jobs table auto-refreshes. Click any job row to open full output inline.

4. Update Schedules

Under System Updates → Schedules:

  • Create a schedule name
  • Enter a cron expression
  • Select target servers and package manager
  • Click Create Schedule

Schedules run in the timezone assigned at creation (based on your active timezone setting at that time).

Examples:

0 2 * * *      # daily at 02:00
0 */6 * * *    # every 6 hours
30 3 * * 0     # every Sunday at 03:30

Use Enable/Disable or Delete to manage schedules.

5. Alerts

When update jobs fail, the system creates alerts.

  • View alerts in Alerts
  • Filter by level and state (open/acknowledged/all)
  • Use Acknowledge to mark an alert as handled
  • Use Dismiss to remove an alert

Admin users also see an open-alert count badge in the sidebar/top bar.

6. Audit Log

Audit Log records key user/system actions and configuration changes.

  • Search by text
  • Filter by action and user
  • Filter by date range
  • Use pagination controls to browse older entries

7. User Management (Admin)

In User Management admins can:

  • Create users
  • Edit profile/admin/enabled status
  • Delete users

Safety rules prevent removing the last enabled admin or disabling/deleting your own active admin account in unsafe ways.

8. SSH Keys (Admin)

In SSH Keys admins can store private keys or generate new keys for server authentication.

Keys in use by servers cannot be deleted until unassigned.

9. Global Settings (Admin)

Global Settings control system-wide behavior:

  • Date/time display format
  • Default timezone
  • History retention (jobs, audit logs, alerts)
  • Login timeout (minutes of inactivity)
  • Email alert enable/disable
  • SMTP server credentials and sender settings
  • SMTP connection test

10. User Settings

Each user can set personal date/time format and timezone, or choose to inherit global defaults.

11. Email Alert Delivery

When email alerts are enabled, alert messages are sent to enabled admin users that have an email address configured in User Management.

12. Common Failure Checks

  • Authentication failed: verify username/key/password on the server
  • Sudo failed: verify sudo password or sudo permissions
  • Package manager unknown: host may not be apt/dnf/yum based
  • Timeout/unreachable: verify network, DNS, firewall, and host status
  • SMTP test failed: verify SMTP host, port, TLS, credentials, and firewall access
