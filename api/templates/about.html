Daygle Server Manager

Daygle Server Manager is a self-hosted web application for remotely patching and maintaining Linux servers over SSH. It provides a central dashboard to trigger package manager updates, schedule recurring maintenance windows, track job outcomes, and receive email alerts when something goes wrong.

The application is built with Python (FastAPI) and deployed exclusively via Docker Compose. All configuration lives in a single daygle_server_manager.conf file — no per-container environment files or secrets scattered across the project.

Key Features

  • SSH-based updates — Connect to any Linux server using stored SSH keys and run package manager updates (apt, dnf, yum, or auto-detected).
  • Scheduled updates — Define cron-based schedules with per-schedule timezone support to automate recurring patch windows across your fleet.
  • Job history — Every update run is recorded with its status, output summary, and timestamps for full auditability.
  • Alerts — Failed jobs automatically generate alerts visible in the UI. Optionally configure SMTP to email all admin users when an alert is raised.
  • Audit log — Administrative actions such as server changes, user management, and settings updates are logged for traceability.
  • SSH key management — Generate or import SSH key pairs directly in the UI. Public keys can be copied and deployed to target servers without leaving the interface.
  • Multi-user support — Create multiple user accounts with optional admin privileges. Per-user date format and timezone preferences are supported.

Technology Stack

  • API — Python 3, FastAPI, SQLAlchemy, Jinja2
  • Worker — Python background process handling SSH update jobs and schedule execution
  • Database — PostgreSQL
  • Deployment — Docker Compose

Getting Started

After the initial setup wizard (/setup), the recommended workflow is:

  1. Go to SSH Keys and generate or import an SSH key pair.
  2. Copy the public key to each target server's ~/.ssh/authorized_keys.
  3. Add your servers under Servers.
  4. Trigger a manual update from System Updates, or create a schedule for automated patching.
