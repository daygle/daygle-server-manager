Daygle Server Manager

Daygle Server Manager is a self-hosted web application for remotely patching and maintaining Linux servers over SSH. It provides a central dashboard to trigger package manager updates, schedule recurring maintenance windows, track job outcomes, and receive email alerts when something goes wrong.

The application is built with Python (FastAPI) and deployed exclusively via Docker Compose. All configuration lives in a single daygle_server_manager.conf file and avoids scattered environment files and secrets.

Key Features

  • SSH-based updates: Run package manager updates using stored SSH credentials (apt, dnf, yum, or auto-detected).
  • Scheduled updates: Define cron-based schedules with timezone support for recurring maintenance windows.
  • Job history: Track run status, output summary, and timestamps for every update operation.
  • Alerts: Failed jobs create alerts in the UI and can optionally send SMTP notifications to admin users.
  • Audit log: Records administrative and configuration actions for traceability.
  • SSH key management: Generate or import key pairs directly in the UI.
  • Multi-user support: Manage multiple users with optional admin privileges and per-user display preferences.

Technology Stack

  • API - Python 3, FastAPI, SQLAlchemy, Jinja2
  • Worker - Python background process handling SSH update jobs and schedule execution
  • Database - PostgreSQL
  • Deployment - Docker Compose

Getting Started

After the initial setup wizard (/setup), the recommended workflow is:

  1. Go to SSH Keys and generate or import an SSH key pair.
  2. Copy the public key to each target server's ~/.ssh/authorized_keys.
  3. Add your servers under Servers.
  4. Trigger a manual update from System Updates, or create a schedule for automated patching.
