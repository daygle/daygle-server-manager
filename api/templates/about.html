About Daygle Server Manager

Overview

Daygle Server Manager is a self-hosted web application for remotely patching and maintaining Linux servers over SSH.

It provides a central control panel to trigger package manager updates, define recurring maintenance schedules, track job history, and receive notifications when something needs attention — all without requiring direct shell access to each server.

The application is built with Python and FastAPI, deployed via Docker Compose. All configuration is stored in a single daygle_server_manager.conf file, keeping setup straightforward and portable.

For step-by-step usage instructions, see the Help page.

Key Features

  • SSH-based updates — run package manager updates on remote servers using stored credentials. Supports apt, dnf, yum, and auto-detection.
  • Scheduled updates — define cron-based schedules with full timezone support for fully automated, recurring maintenance windows.
  • Job history — every update operation is logged with status, output, and timestamps for a complete audit trail.
  • Alerts — failed jobs and post-update reboot requirements generate alerts in the UI. Optional SMTP delivery notifies admins by email.
  • Audit log — records all administrative actions, configuration changes, and system events for traceability and compliance.
  • SSH key management — generate or import SSH key pairs directly in the UI. Private keys are encrypted at rest.
  • File explorer — browse, view, edit, upload, and download files on remote servers over SFTP without leaving the browser.
  • SSH terminal — run ad-hoc commands on a server via an in-browser terminal session.
  • Server status — monitor CPU, RAM, storage, and load across all configured servers at a glance.
  • Multi-user support — manage multiple users with admin or standard roles, per-user display preferences, and individual timezone settings.

Technology Stack

  • API — Python 3, FastAPI, SQLAlchemy, Jinja2
  • Worker — Python background process managing SSH update jobs and cron schedule execution
  • Database — PostgreSQL
  • Deployment — Docker Compose
